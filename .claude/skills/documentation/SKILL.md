# Documentation Standards

## README Requirements

1. What it does (one paragraph)
2. Quick start / installation
3. Usage examples
4. Configuration options
5. Support section with links

## ADRs

Add to `specs/decisions.md` for significant technical decisions.

## Code Comments

- Comment the WHY, not the WHAT
- Document non-obvious decisions inline
- Keep API docs updated with code changes
